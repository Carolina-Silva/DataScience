# Engenharia de Dados

## O que é Engenharia de Dados?
A **Engenharia de Dados** é a área responsável por **desenvolver, manter e operar sistemas que coletam, organizam, armazenam e processam dados**. Seu principal objetivo é **preparar e disponibilizar dados de forma confiável e escalável** para que áreas como **Ciência de Dados, Analytics e BI** possam extrair valor.

Em outras palavras, a engenharia de dados cria a **base** que sustenta a ciência de dados.

---

## Importância do Conhecimento sobre Tipos de Dados
Para trabalhar com dados (especialmente em Ciência de Dados), é fundamental conhecer os **diferentes tipos de dados** existentes, pois eles influenciam:
- A forma de armazenamento
- As ferramentas utilizadas
- As técnicas de processamento e análise

---

## Ciclo de Vida dos Dados
O ciclo de vida dos dados descreve todas as etapas pelas quais os dados passam dentro de uma organização.

### ETL
- **E (Extract)**: extração dos dados de diferentes fontes
- **T (Transform)**: limpeza, padronização e transformação dos dados
- **L (Load)**: carregamento dos dados em um destino (data warehouse, data lake, etc.)

A engenharia de dados está diretamente ligada a esse processo.

---

## Engenharia de Dados e Big Data
A **Engenharia de Dados** está fortemente relacionada à **movimentação e ao gerenciamento de bases enormes de dados**.

### Big Data
O termo **Big Data** surgiu em 1998 para definir o crescimento acelerado na:
- Produção
- Armazenamento
- Processamento de dados

#### Os 3 Vs do Big Data
- **Volume**: grande quantidade de dados
- **Velocidade**: geração e processamento rápidos
- **Variedade**: diferentes formatos e fontes de dados

---

## Armazenamento de Dados
O **armazenamento de dados** é uma das principais responsabilidades do engenheiro de dados.

A evolução do **hardware** (discos, memória, processamento distribuído) foi essencial para permitir:
- A criação
- O armazenamento
- O processamento de grandes volumes de dados

---

## Engenheiro de Dados vs Cientista de Dados

### Engenheiro de Dados (Infraestrutura)
- Cuida dos sistemas de **coleta de dados**
- Realiza **migração de dados**
- Trabalha com **diferentes tipos de dados**
- Organiza e mantém a **infraestrutura de dados**

### Cientista de Dados (Exploração)
- Cria **modelos estatísticos e de machine learning**
- Elabora **dashboards** e análises
- Define e acompanha **KPIs**
- Desenvolve soluções baseadas em dados

Ambos dependem de uma **base de dados bem estruturada**.

---

## Níveis de Maturidade dos Dados
Os **níveis de maturidade dos dados** representam o grau de:
- Desenvolvimento
- Gerenciamento
- Uso
- Governança de dados

Dentro de uma organização.

---

## Bancos de Dados e SQL

### MySQL
- Banco de dados **relacional**
- Geralmente utilizado localmente ou em servidores
- Usado para armazenar dados estruturados

### SQL
- **Structured Query Language**
- Linguagem padrão para:
  - Consultar
  - Manipular
  - Definir dados em bancos relacionais

### Bancos de Dados Relacionais
- Os dados são organizados em **tabelas**
- As tabelas se relacionam entre si por meio de **chaves**

---

## Consumidores e Criadores de Dados

Estamos constantemente **criando e consumindo dados**, por exemplo:
- Redes sociais
- Sistemas corporativos
- Aplicações digitais

Esses dados podem ser usados por diversas pessoas e áreas, de formas diferentes.

- Para alguns problemas, **grandes volumes de dados** são essenciais
- Para outros, **poucas informações bem selecionadas** são suficientes

A quantidade de dados influencia diretamente a **estrutura necessária** para armazená-los — em alguns casos, uma planilha (Excel) já é suficiente.

---

## Diversidade das Necessidades de Dados
Uma mesma empresa pode ter um grande conjunto de dados, mas:
- Cada time precisa de **subconjuntos diferentes**
- Os dados devem ser organizados de forma flexível e acessível

---

## Sistemas Operacionais vs Sistemas Analíticos

### Sistemas Operacionais
- Servidores de backend
- Infraestrutura
- Aplicações
- Sistemas que **criam e modificam dados**

### Sistemas Analíticos
- Utilizados por analistas de negócio e cientistas de dados
- Focados em:
  - Análise
  - Relatórios
  - Modelagem

Ambos fazem uso da **mesma base de dados**, porém com objetivos diferentes.

O engenheiro de dados deve saber **integrar sistemas operacionais e analíticos** em um único ambiente de dados.

---

## Chaves em Banco de Dados

- **PK (Primary Key)**: identifica unicamente um registro em uma tabela
- **FK (Foreign Key)**: cria o relacionamento entre tabelas

Essas chaves são fundamentais para garantir **integridade e relacionamento dos dados**.
